# Michelin 餐廳評論使用者洞察摘要
### 以資料科學視角重新理解「米其林評價」與「真實消費者體驗」之間的差距

本專案透過**881則Yelp用戶評論**，結合**情感分析（Sentiment Analysis）、主題模型（LDA Topic Modelling）、統計驗證（t-test、ANOVA、迴歸**等方法，探討**米其林評鑑（1–3星級 & Bib Gourmand）是否真正反映一般消費者的體驗**，並深入萃取影響顧客滿意度的關鍵因素。

專案目標不僅是資料分析，更在於將數據轉化為「可被餐飲品牌、決策者與PM採用的使用者洞察與可行策略」，協助理解價值感、體驗落差與優化方向。

## 1. 專案背景
米其林餐廳被視為餐飲品質指標，但大多數消費者實際選餐廳是依賴：
- Yelp
- Google Maps
- TripAdvisor
- etc.

這產生一個關鍵問題：
> **米其林星級的「專家評鑑」是否真的反映一般用戶的實際感受？**
> 如果不一致，差異來自哪裡？餐廳應該如何改善體驗？

## 2. 目標
#### 研究目標
- 分析米其林星級、Bib Gourmand 與真實使用者評價之間的關係
- 找出影響消費者「滿意/不滿意」的關鍵體驗元素
- 建立可用於品牌決策、產品化、體驗設計的使用者洞察模型
#### 作品集/PM 能力目標
- 展示我如何把複雜資料 → 可行洞察
- 展示問題拆解、分析與決策能力
- 展示資料思維如何支援產品/服務改善


## 3. 我的角色
在這個專案中，我負責：
- 需求定義（要解決什麼問題、誰在乎）
- 資料蒐集策略（Yelp + Michelin）
- 研究與分析方法選擇（Sentiment、LDA、ANOVA）
- 把資料結果翻成「可行的產品/體驗策略」
- 統整成最終洞察摘要、報告與視覺化

## 4. 專案方法
#### 我運用：
- 881 则 Yelp 评论資料清理與前處理
- 情感分析（Sentiment Analysis）：判斷每則評論的正負向
- 主題模型（LDA）：找出消費者最常提到的體驗重點
- 統計檢定（t-test、ANOVA）：比較不同星級、價格、評論群之差異
- 回歸分析（Regression）：檢驗價格、星級、情緒的關係

#### 流程架構：
- 定義問題與假設（例如：價格越高，使用者越滿意？）
- 資料選擇與清洗
- 驗證情緒與評價的關聯
- 比較星級 vs Bib Gourmand 差異
- 分析負評根因
- 輸出策略建議（適合 PM、餐飲品牌採用）


## 5. 關鍵洞察（User Insights）
洞察 1｜米其林餐廳整體滿意度高，但負評來自「期望落差」
Yelp 上 80% 為 4–5 星 → 消費者普遍滿意
但負評語氣強烈，多為：
「沒有達到我期待的星級體驗」
「這價位應該更好」
👉 米其林星級本身提高期待，也提高了失望的風險。
---
洞察 2｜高價 ≠ 高滿意度：消費者重視「價值感」，不是價格
價格與滿意度呈 顯著負相關
最高價餐廳反而得到較低的評分
常見負評：
overpriced
expected more
not worth it
👉 真正的痛點是「價格與體驗的落差」。
---
洞察 3｜Bib Gourmand 比星級餐廳更受一般消費者喜愛
平均分數 4.49（明顯高於星級的 4.24）
評價更穩定、負評更少
常出現字詞：
worth it
friendly
would return
👉 高 CP 值餐廳最能滿足大眾用戶需求。
---
洞察 4｜驅動好評的是「食物 + 服務」，驅動負評的是「流程 + 價值」
正評高頻字詞：
delicious、amazing、service、staff
負評高頻字詞：
disappointed、money、slow、arrived、worst
👉 食物決定上限、服務決定下限、價值決定是否會回訪。
---
洞察 5｜特殊場合體驗是推動品牌的最大槓桿，也最容易出錯
LDA 顯示許多評論與「生日、紀念日、下午茶」相關
若體驗好 → 評分極高
若體驗壞 → 評分極低
👉 特殊場合是最敏感的旅程節點，需額外設計。
